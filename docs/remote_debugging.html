Remote Debugging

To perform debugging between two different machines you must use the Debug->Connect menu command.

When it opens you can select the remote machine address as well as the port and password. On the machine running Torsion you must have an identical set of script files to the ones running on the local machine, else you'll get missing file errors and incorrect line numbers. Other than that the local machine is ready.

On the Remote Machine

The remote machine needs to have your game already running and the TelnetDebugger enabled. To do this you must either add the following code into one of your scripts, preferably your main entry script, or enter that command into the console...

dbgSetParameters( 6060, "password", true );

Now be sure that the port and password match the settings in Torsion. The final true value tells the game to stop and wait until Torsion connects. If that is set to false the game will continue running until Torsion connects to it.

Ready to Debug

Your now ready to debug your remote machine, so hit the Connect button in dialog. If everything worked correctly you should now be in the debugger where everything works just like it was a local debug session.
